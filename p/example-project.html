Example Project — à remplacer

Entrée de démonstration : elle existe pour vérifier que le pipeline JSON → page statique → catalogue → sitemap fonctionne avant le tout premier vrai projet catalogué.

Ce fichier existe uniquement pour que build.js ait quelque chose à traiter au tout premier build, et pour que tu voies à quoi ressemble une page générée (corps en Markdown, visuel, tags) avant de connecter un vrai dépôt.

Comment ça marche

Le corps de chaque entrée est du Markdown restreint, rendu par un vrai moteur (marked) des deux côtés — build et aperçu navigateur. Ça veut dire que du code recopié depuis un vrai README s'affiche correctement :

npm install mon-projet
npm run build

Une fois un token GitHub et un fournisseur IA configurés depuis le panneau d'administration, génère une vraie entrée à partir d'un de tes dépôts, puis supprime purement et simplement ce fichier projects/example-project.json.

Stack

Cette entrée de démonstration n'a pas de vraie stack — mais tes vrais projets, si le README la précise, en auront une ci-dessous.
